# Design: Resolve CC0 / MIT license mismatch

- **Date:** 2026-05-22
- **Status:** Approved (ready for implementation plan)
- **Decision:** CC0-1.0 is the canonical license. Correct the stale MIT
  references and make the built `.deb` actually declare CC0.

## Problem

The repository declares two conflicting licenses. The canonical
`LICENSE` file and the README were switched to CC0 1.0 Universal on
2026-05-17 (commit `e1b75a1` "CC0 licensing"), but two older files were
left declaring MIT:

| Location | Current value | Added |
| --- | --- | --- |
| `LICENSE` (full text) | **CC0 1.0 Universal** | 2026-05-17 (`e1b75a1`) |
| `README.md:183` | **CC0 1.0 Universal** | 2026-05-17 (`e1b75a1`) |
| `NOTICE.md:3` | **MIT** | 2026-05-15 (`e8c9832`) |
| `packaging/nfpm.yaml:13` (`license:`) | **MIT** | 2026-05-05 (`f3c3a61`) |

A whole-tree word-boundary sweep confirms these are the **only** two MIT
references outside the planning docs (`SPEC*.md`, `PHASE*.md`, which use
"MIT" only inside unrelated words). No Go source file carries a license
or `SPDX-License-Identifier` header.

### The "Debian package" nuance

There is no `debian/` directory; the `.deb` is generated by nfpm
(`make deb` → `nfpm pkg --packager deb`). nfpm's `license:` field is
**documentary only for deb output** — it is not written into the deb
control file and nfpm does not generate a `debian/copyright` file from
it (it feeds rpm/apk metadata only). Consequently:

- Changing `nfpm.yaml`'s `license:` field corrects the source-of-truth
  metadata but does **not**, by itself, make the built `.deb` declare
  any license.
- The current `.deb` ships **no** copyright file at all (its `contents:`
  list only the binary, the `/etc` dir, the systemd unit, and the
  default config). That is a Debian Policy gap (lintian
  `no-copyright-file`).

To make the Debian package genuinely declare CC0, the package must ship
a copyright file at the Debian-conventional path
`/usr/share/doc/apt-cacher-ultra/copyright`.

## Decision

Adopt **Approach B**: correct the two MIT references *and* ship a
machine-readable Debian copyright file declaring CC0-1.0.

Rejected alternatives:

- **A — text-only correction.** Edit only `NOTICE.md` and `nfpm.yaml`.
  Rejected because it leaves the built `.deb` still shipping no license
  file, so the "Debian package" the user named would not actually
  declare CC0.
- **C — full SPDX/REUSE.** Add `SPDX-License-Identifier: CC0-1.0`
  headers to every Go source file (and/or adopt REUSE). Rejected as
  out of scope: it touches every source file and goes well beyond the
  reported mismatch. May be revisited later if license scanners are
  needed.

## Changes

1. **`NOTICE.md:3`** — replace the MIT line with CC0 wording consistent
   with the README's "Released into the public domain under CC0 1.0
   Universal":

   ```
   apt-cacher-ultra is released into the public domain under the
   [CC0 1.0 Universal](LICENSE) dedication.
   ```

2. **`packaging/nfpm.yaml:13`** — `license: MIT` → `license: CC0-1.0`
   (SPDX short identifier).

3. **New file `packaging/copyright`** — Debian machine-readable (DEP-5)
   format:
   - Header: `Format:` (DEP-5 spec URL), `Upstream-Name:
     apt-cacher-ultra`, `Upstream-Contact: Sean Reifschneider
     <jafo00@gmail.com>`, `Source: https://github.com/linsomniac/apt-cacher-ultra`.
   - `Files: *` stanza → `Copyright: 2026 Sean Reifschneider
     <jafo00@gmail.com>`, `License: CC0-1.0`, with the **full CC0 1.0
     text embedded inline** (each line space-indented; blank lines
     written as ` .` per DEP-5). The license content is taken verbatim
     from the existing `LICENSE` file (re-indented for DEP-5 as above).
     Inline text is required because CC0-1.0 is **not** present in
     `/usr/share/common-licenses`, so a bare reference would not be
     lintian-clean.
   - `Files: internal/gpg/embedded/*` stanza → documents the bundled
     archive keyrings as redistributable public trust material
     (Canonical Ltd. / Debian Project), cross-referencing `NOTICE.md`
     for sources and verification guidance.

4. **`packaging/nfpm.yaml` `contents:`** — add an entry installing the
   copyright file:

   ```yaml
     - src: ./packaging/copyright
       dst: /usr/share/doc/apt-cacher-ultra/copyright
       file_info:
         mode: 0644
         owner: root
         group: root
   ```

5. **`e2e/deb/test.sh`** — extend the existing package-contract checks:
   - Add an existence check alongside the others (~line 49):
     `test -f /usr/share/doc/apt-cacher-ultra/copyright`, plus
     `grep -q 'CC0-1.0' /usr/share/doc/apt-cacher-ultra/copyright`.
   - Add `[/usr/share/doc/apt-cacher-ultra/copyright]="644"` to the
     `expected_mode` associative array so the exact-mode assertion
     covers it.

## Testing & verification

- `make deb` (requires nfpm) builds without error.
- `dpkg-deb -c build/apt-cacher-ultra_*_amd64.deb` lists
  `/usr/share/doc/apt-cacher-ultra/copyright`.
- `make deb-test` (or the `e2e/deb` harness) passes, including the new
  copyright assertions.
- Re-run the MIT word-boundary sweep; expect zero matches outside
  `SPEC*.md` / `PHASE*.md`.

## Trade-offs & notes

- The CC0 text will exist in both `LICENSE` and `packaging/copyright`.
  This duplication is intentional and low-risk — CC0 1.0 is fixed,
  standard legal text — and is the boring/maintainable choice versus
  build-time generation of the copyright file.
- `CONTRIBUTING.md` refers only to "the project's existing license"
  generically (no MIT/CC0 name), so it needs no change.

## Out of scope

- SPDX headers in source files / REUSE compliance (Approach C).
- Any change to the bundled keyrings themselves or their refresh
  workflow.
